# Mental Health Screening Tool

Веб-приложение для психиатрических клиник, позволяющее пациентам пройти стандартизированные психологические тесты перед визитом к психиатру. Целевая аудитория — пациенты и врачи-психиатры.

---

## Tech Stack

- **Frontend:** Vanilla JavaScript (ES6+), HTML5, CSS3 (Grid, Flexbox, CSS Variables)
- **Шрифты:** Google Fonts — Spectral (заголовки), DM Sans (текст)
- **Экспорт:** html2canvas (загружается асинхронно с CDN, graceful fallback)
- **Scripting:** Python 3 — вспомогательный скрипт для генерации идей через OpenAI API
- **AI:** OpenAI API (модель `gpt-4o-mini`), библиотека `openai`, `python-dotenv`
- **Сборка:** отсутствует — никаких bundler'ов, фреймворков, npm/yarn

## What Claude Should Know

- **Архитектура:** приложение намеренно сделано как один HTML-файл без зависимостей — это требование, а не упущение. Не предлагать переезд на фреймворки.
- **Конфиденциальность:** все данные остаются на стороне клиента (in-memory). Нет серверных запросов, нет localStorage (пока). Это принципиально для медицинских данных.
- **Тесты — валидированные инструменты:** PHQ-9, GAD-7, MDQ, AUDIT, PSS-10 — стандартные клинические опросники с фиксированными вопросами, шкалами и пороговыми значениями. Изменять их содержание и логику подсчёта нельзя без клинического обоснования.
- **MDQ имеет нестандартную логику подсчёта:** 3 части с разными правилами (часть 1 — 13 бинарных ответов, часть 2 — да/нет, часть 3 — тяжесть 0–3). Учитывать это при любых изменениях.
- **Состояние приложения:** управляется через переменные `CT` (текущий тест), `QI` (индекс вопроса), `ANS` (ответы), `SV` (выбранное значение), `SAV` (сохранённые результаты). Это in-memory — при перезагрузке страницы данные теряются.
- **Экспорт в PNG:** зависит от html2canvas, загружаемого с CDN. При недоступности CDN показываются инструкции по ручному скриншоту.
- **Скрипт `ask_ai.py`:** вспомогательный инструмент для генерации идей, не является частью продукта.

## Don't

- **Не добавлять серверную часть или внешние API в HTML-файл** — приложение намеренно полностью клиентское. Передача медицинских данных на сервер требует отдельного архитектурного решения и согласования.
- **Не изменять вопросы, веса и пороговые значения** в тестах PHQ-9, GAD-7, MDQ, AUDIT, PSS-10 без клинического обоснования — это медицинские инструменты со стандартизированными параметрами.
- **Не добавлять build-процесс, npm, bundler или фреймворки** без явного запроса — простота единого HTML-файла является архитектурным решением, а не техническим долгом.
- **Не хранить OPENAI_API_KEY в коде** — только через `.env`, который исключён из git.
- **Не добавлять persistent storage (localStorage/база данных)** для медицинских данных без предварительного обсуждения — это влечёт требования к конфиденциальности и безопасности данных.
